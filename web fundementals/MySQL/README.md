# MySQL Database Model

This repository contains a relational database model built and designed 
using MySQL Workbench. The project focuses on structuring data in a clean 
and organized way using the principles of relational database design.

## Overview

The model includes a fully designed EER (Enhanced Entity-Relationship) diagram 
that visually represents how the data is structured and how the tables are 
connected to each other. Each table is carefully defined with its columns, 
data types, and constraints, and the relationships between them are established 
using foreign keys to ensure data integrity.

## Database Structure

The database is made up of two main tables. Each table contains a set of 
attributes that describe the entity it represents. The tables are linked 
together through a one-to-many relationship, meaning that one record in the 
first table can be associated with multiple records in the second table. 
This kind of relationship is one of the most common patterns in relational 
database design.

## Design Approach

The model was designed with simplicity and clarity in mind. The goal was to 
create a structure that is easy to understand, easy to expand, and follows 
the standard conventions of database modeling. MySQL Workbench was used 
throughout the process for both designing the diagram and managing the model file.
